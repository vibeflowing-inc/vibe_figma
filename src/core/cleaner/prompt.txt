## 1. CSS & Layout Cleanup (Tailwind) - MANDATORY FIXES

### Remove Scraper Artifacts (Machine-Generated "Junk")
Strip out ALL machine-generated artifacts and useless classes:
- `origin-[...]` - Remove all transform origin classes
- `bg-repeat`, `bg-scroll` - Remove background repeat/scroll defaults
- `text-clip` - Remove text clipping artifacts
- `box-border` - Remove unless specifically needed
- `z-index` abuse - Remove excessive z-index stacking
- Overly specific pixel values like `h-[1875.39px]`, `w-[1012px]`, `left-[16px]`, `top-[24.5px]`
- `opacity-100`, `visible`, `static` - Remove default value classes
- Duplicate or contradicting classes (e.g., `flex` and `block` together)

### Fix Positioning Issues
- **Remove ALL `absolute`, `fixed`, and `inset` positioning used for layout**
- Replace with **responsive Flexbox and Grid** layouts
- Use `relative` only when actually needed for child positioning
- Convert fixed positioning hacks to proper layout solutions

### Standardize Colors (PRESERVE EXACT VISUAL APPEARANCE)
Replace raw color format with Tailwind while keeping EXACT color values:
- `oklch(...)` → Standard Tailwind colors ONLY if EXACT match (e.g., `bg-neutral-900`)
- `rgba(...)` → Tailwind opacity utilities ONLY if EXACT match (e.g., `bg-black/50`)
- `#000000` → `text-black`, `bg-black` ONLY if pure black
- `#ffffff` → `bg-white` ONLY if pure white
- **IMPORTANT:** If color is not a perfect match to Tailwind preset, KEEP the arbitrary value `bg-[#f5f5f5]`
- **NEVER approximate colors** - `#f5f5f5` is NOT the same as `bg-gray-100`
- When in doubt, keep the original color value to preserve visual accuracy

### Make Fully Responsive (WHILE KEEPING SAME VISUAL APPEARANCE)
- Replace fixed pixel widths with responsive alternatives that maintain the same maximum size:
  - `w-[1440px]` → `max-w-7xl mx-auto` (7xl = 80rem = 1280px, use `max-w-[1440px]` if closer match needed)

- Replace fixed heights with flexible alternatives ONLY when safe:
  - `h-[600px]` on containers → `min-h-[600px]` or `h-auto` (only if content-driven)
  - Keep fixed heights on hero sections, image containers, or design-critical elements
- Add responsive breakpoint classes for mobile/tablet ONLY when needed
- **IMPORTANT:** Default desktop view must look IDENTICAL to original
- Images must scale: `max-w-full h-auto` or `w-full object-cover` (preserve aspect ratio)

### Consolidate Classes
- `m-2` instead of `mt-2 mr-2 mb-2 ml-2`
- `p-4` instead of `pt-4 pr-4 pb-4 pl-4`
- Use shorthand whenever possible
